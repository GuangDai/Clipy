# ClipyApp

The macOS menu-bar application (composition root) and its hosted
`ClipyIntegrationTests` bundle, generated by XcodeGen.

- The **app target is XcodeGen-owned**; the **library graph is SwiftPM-owned**
  (root `Package.swift`) and consumed here as the local package `Clipy`
  (docs/01-architecture.md §9 item 6).
- Step-0 scaffold only: the real composition root (concrete construction,
  lifecycle, paste orchestration, dependency injection) lands at roadmap step 9b
  (docs/roadmap/06-clipyapp.md).

## Generate and open (macOS 26 / Xcode 26 runner)

```sh
# from the repository root
xcodegen generate --spec ClipyApp/project.yml
open ClipyApp/ClipyApp.xcodeproj
```

Select the `ClipyApp` scheme: ⌘R runs the agent stub (`LSUIElement`, a single
"Clipy — scaffold" placeholder window); ⌘U runs `ClipyIntegrationTests` hosted
by the app.

The generated `.xcodeproj` is a build artifact — regenerate it after any change
to `project.yml` instead of editing it in Xcode.
